SOLUTION to display images uploaded by user:
Step 1:
As per [1], images uploaded by users should be served so as to be displayed:
Edit webplatform/urls.py, add '+ static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)' to the urlpatterns list after doing some corresponding importing.
The code snippet can be found in [1]. Notice that this is only for developing stage of our MIPP.
Step 2:
In pages need to display images uploaded by users, { % load static % } at the top of the page instead of { % load staticfiles % } as you previously do.
At last, modify the src attribute in img tag to be { % get_media_prefix % }{ { your_image_path } } instead of { % static your_image_path % }
PS:
Images in http://www.mipp.tech/datamanagement/dataset_act can be now displayed normally.
Open a console shell in pythonanywhere, cd to ~/www.mipp.tech and type 'git diff' to see the differences of the source code.
